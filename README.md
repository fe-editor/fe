# fe

> _fe_ is a recursive acronym that stands for "fe editor".

## Goals

In no particular order...

1. Modal editing with a focus on ergonomics/comfort
2. Extendability with scripting (Scheme, or potentially other embeddable 
   language options)
3. Intuitive and composable commands
4. The rest are yet to be clarified... 

## MVP

1. Accomplish basic working text edit interface
2. Use prototype to test and improve UX
3. Go fast
4. Go -> TUI, Zig -> editor core
5. Refactor/improve during development if necessary
